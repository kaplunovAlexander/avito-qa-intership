## Общая информация

### Объект тестирования
Микросервис объявлений:
- создание объявления
- получение объявления по идентификатору
- получение всех объявлений продавца
- получение статистики по item id

### Базовый URL
https://qa-internship.avito.com

# 1. POST /api/1/item — Создание объявления

## TC-POST-001
Создание объявления с валидными данными  
Ожидаемый результат: 200 OK, возвращается объект с уникальным id, поля соответствуют переданным.

## TC-POST-002
Создание объявления с одинаковыми данными дважды  
Ожидаемый результат: создаются два объявления с разными id.

## TC-POST-003
Создание объявления без sellerID  
Ожидаемый результат: 400 Bad Request.

## TC-POST-004
Создание объявления без name  
Ожидаемый результат: 400 Bad Request.

## TC-POST-005
Создание объявления без price  
Ожидаемый результат: 400 Bad Request.

## TC-POST-006
Создание объявления с sellerID строкового типа  
Ожидаемый результат: 400 Bad Request.

## TC-POST-007
Создание объявления с price строкового типа  
Ожидаемый результат: 400 Bad Request.

## TC-POST-008
Создание объявления с отрицательной ценой  
Ожидаемый результат: 400 Bad Request.

## TC-POST-009
Создание объявления с price = 0  
Ожидаемый результат: корректная обработка граничного значения.

## TC-POST-010
Создание объявления с отрицательными значениями статистики  
Ожидаемый результат: 400 Bad Request.

## TC-POST-011
Создание объявления с очень длинным name  
Ожидаемый результат: корректная обработка или 400, без 500 ошибки.

## TC-POST-012
Создание объявления с лишними полями  
Ожидаемый результат: лишние поля игнорируются или 400.

# 2. GET /api/1/item/{id} — Получение объявления

## TC-GET-ID-001
Получение существующего объявления  
Ожидаемый результат: 200 OK, данные совпадают с созданными.

## TC-GET-ID-002
Получение объявления по несуществующему id  
Ожидаемый результат: 404 Not Found.

## TC-GET-ID-003
Получение объявления с некорректным id формата string  
Ожидаемый результат: 400 Bad Request.

## TC-GET-ID-004
Проверка структуры ответа  
Ожидаемый результат: присутствуют поля id, sellerId, name, price, statistics, createdAt.

# 3. GET /api/1/{sellerID}/item — Получение объявлений продавца

## TC-GET-SELLER-001
Получение объявлений по валидному sellerID  
Ожидаемый результат: 200 OK, возвращается список объявлений.

## TC-GET-SELLER-002
Проверка принадлежности объявлений sellerID  
Ожидаемый результат: все объявления в списке имеют указанный sellerID.

## TC-GET-SELLER-003
Получение объявлений для sellerID без записей  
Ожидаемый результат: возвращается пустой список.

## TC-GET-SELLER-004
Получение объявлений по sellerID строкового типа  
Ожидаемый результат: 400 Bad Request.

## TC-STAT-V1-002
Получение статистики по несуществующему id

## TC-STAT-V1-003
Проверка типов данных статистики  
Ожидаемый результат: все поля числовые и неотрицательные.
